Components Of The Framework

The Grid top

The Row Class

The "row" div is the outer div. Inside the "row" you can put as many numbered column div's filled with content as you want.

These child div's must always equal 100% for a complete row, i.e. there must always be 6 div's with the class="six-columns", always 4 div's with the class="four-columns", etc.

Content that is not divided into columns but instead is just one big single column taking up the full width of the container doesn't need to be nested inside any "row" or column div. Keep these items out of a "row" div to make sure the left & right margins are correct and the content is flush on the left & right with the columned content.

Forms top

Forms are already sized appropriately using their own set of flexbox rules, so <form> doesn't need to go inside a "row" div.

Since the form is a flexbox with flex-direction: column, each item will split onto it's own line. Putting the checkbox and radio buttons (with their descriptive text next to them) into a <div> will keep them as one item and therefore on the same line.

Example:

<div>
  <input type="checkbox"> Hello There
</div>

Putting multiple checkboxes or radio buttons in one <div> will keep them all inline. For separate lines, put each checkbox or radio button into it's own <div>

Likewise, if I want a Submit and Reset button to be on the same line they need to be put together in one <div>

Just a reminder about radio buttons: they need to have the same name="" so that way only one can be selected at a time. Different names, or just no names entered, will leave all the radio buttons selectable at the same time.

One Two

For bigger screens the form has a max width of 500px. If I want the form to be inside one column and text to be inside a second column next to it (both of these columns inside a "row"), the form's padding, specifically the padding-top, will need to be removed so that the top of the columns, form and text, are both flush. I won't be making a utility class for this, it's something that can be added per site in it's custom css.

When it's a search bar make sure to include the id="search" in your input, the styling targets that id. The form needs to have the class search-bar added to it. If your want a search button using an icon add the class search-btn-icon, if you want the word "search" then add the class search-btn-word to your button.

Inline form for username and passwords. The form has the class inline-form, each label has the class label-inline, and each input has the class input-inline. And the button has the class search-btn-word. No testing has been done to see how this would fare inside a colum.

Utility Classes top

These utility classes are helpers for specific instances.

float-lt
Adds float: left to whatever element it's added to

float-rt
Adds float: right to whatever element it's added to

align-rt
Adds align-self: flex-end to whatever flex element it's added to

align-lt
Adds align-self: flex-start to whatever flex element it's added to

text-lt
Adds text-align: left to whatever element it's added to

text-rt
Adds text-align: right to whatever element it's added to

text-center
Adds text-align: center to whatever element it's added to

clear
Adds clear: both to whatever element it's added to

margins-0
Adds margin: 0 to whatever element it's added to
